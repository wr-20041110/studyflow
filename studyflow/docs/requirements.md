# StudyFlow 需求文档

## 项目概述

**项目名称**: StudyFlow 学习任务管理系统
**项目描述**: 一个面向学生的学习任务管理系统，支持个人任务管理与学习进度跟踪
**目标用户**: 学生
**协作时间**: 2026-05-20
**实验课程**: 软件工程 - Vibe Coding 实验

---

## 1. 用户角色

| 角色 ID | 角色名称 | 角色描述 | 权限 |
|---------|----------|----------|------|
| UR-01 | 学生 | 使用系统进行学习任务管理和进度跟踪的个人用户 | 创建/查看/更新/删除自己的任务、查看进度报告、设置提醒 |

---

## 2. 用户故事

### US-01: 创建学习任务

**作为**一名学生，  
**我想要**创建一个新的学习任务并设置其属性（标题、描述、优先级、截止日期），  
**以便于**我可以明确自己的学习目标和计划。

**验收标准**:
- 用户可以输入任务标题（必填）
- 用户可以输入任务描述（选填）
- 用户可以选择任务优先级：高/中/低（必填）
- 用户可以设置截止日期（选填，但高优先级任务必填）
- 创建成功后返回任务ID和完整任务信息

---

### US-02: 更新任务状态

**作为**一名学生，  
**我想要**更新任务的状态（未开始/进行中/已完成），  
**以便于**我可以跟踪自己的学习进度。

**验收标准**:
- 用户可以选择任务状态：未开始/进行中/已完成
- 已完成任务不能被修改为未开始或进行中状态
- 更新成功后返回更新后的任务信息

---

### US-03: 设置任务提醒

**作为**一名学生，  
**我想要**为任务设置提醒时间，  
**以便于**我可以及时完成学习任务。

**验收标准**:
- 用户可以设置任务的提醒时间
- 系统在到达提醒时间时发送通知
- 提醒策略可以基于任务的截止日期

---

### US-04: 查看学习进度报告

**作为**一名学生，  
**我想要**查看指定时间段内的学习进度统计，  
**以便于**我可以了解自己的学习成果和效率。

**验收标准**:
- 用户可以指定查询的时间范围（开始日期和结束日期）
- 系统返回该时间段内的任务统计：总任务数、已完成数、进行中数、未开始数
- 系统计算并显示完成率

---

### US-05: 管理任务优先级

**作为**一名学生，  
**我想要**设置和修改任务的优先级，  
**以便于**我可以合理安排学习顺序，优先完成重要任务。

**验收标准**:
- 用户可以设置任务优先级为高、中、低
- 高优先级任务必须设置截止日期
- 修改优先级时需要满足相关业务规则

## 5. 业务规则（不变量）

| 规则 ID | 描述 | 触发场景 |
|---------|------|----------|
| BR-01 | 任务截止时间不能早于任务创建时间 | 创建任务、更新截止日期 |
| BR-02 | 已完成任务不能再次被修改为未开始或进行中状态 | 更新任务状态 |

## 6. 数据模型

### 任务 (Task)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 任务唯一标识 |
| title | string | 是 | 任务标题 |
| description | string | 否 | 任务描述 |
| status | enum | 是 | 任务状态：NOT_STARTED, IN_PROGRESS, COMPLETED |
| priority | enum | 是 | 优先级：HIGH, MEDIUM, LOW |
| dueDate | datetime | 条件* | 截止日期（高优先级必填） |
| userId | string | 是 | 所属用户ID |
| createdAt | datetime | 是 | 创建时间 |
| updatedAt | datetime | 是 | 更新时间 |

### 进度报告 (ProgressReport)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reportId | string | 是 | 报告唯一标识 |
| userId | string | 是 | 所属用户ID |
| startDate | datetime | 是 | 统计开始时间 |
| endDate | datetime | 是 | 统计结束时间 |
| totalTasks | number | 是 | 总任务数 |
| completedTasks | number | 是 | 已完成任务数 |
| inProgressTasks | number | 是 | 进行中任务数 |
| notStartedTasks | number | 是 | 未开始任务数 |
| completionRate | number | 是 | 完成率（百分比） |

## 8. 验收标准汇总

### AC-01: 任务创建

- [ ] 可以创建包含标题、描述、优先级、截止日期的任务
- [ ] 标题为必填项
- [ ] 优先级为必填项（高/中/低）
- [ ] 高优先级任务必须设置截止日期
- [ ] 返回包含任务ID和完整信息的响应

### AC-02: 任务状态更新

- [ ] 可以将任务状态从未开始改为进行中
- [ ] 可以将任务状态从进行中改为已完成
- [ ] 已完成任务不能改回未开始或进行中状态
- [ ] 返回更新后的任务信息

### AC-03: 进度报告

- [ ] 可以按时间范围查询任务统计
- [ ] 统计包含总任务数、已完成数、进行中数、未开始数
- [ ] 正确计算完成率（已完成数/总任务数 × 100%）
- [ ] 时间范围参数格式正确

### AC-04: 业务规则验证

- [ ] 系统拒绝创建截止时间早于当前时间的任务
- [ ] 系统拒绝将已完成任务的状态改回
- [ ] 系统拒绝创建无截止日期的高优先级任务
