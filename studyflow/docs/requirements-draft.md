# StudyFlow 需求草案（学生版）

## 项目背景

嗯...我想做一个学习任务管理的小系统。因为我总觉得学习的时候记不住哪些任务要做，有时候任务多了就搞混了。就想着搞个东西帮我记一下学习进度什么的。

## 用户角色

其实就我这种人吧，普通学生。就想要个地方记录自己的学习任务，看看自己学得怎么样。

## 我想要什么（大概的想法）

### 关于任务管理

1. **创建任务**
   - 我能新建一个学习任务
   - 填个标题就行（这个必须的）
   - 也可以写点备注说明一下
   - 这个任务重要不重要的，可以选一下（高中低三个档吧）
   - 截止日期也得能设...不过好像不是每个任务都需要截止日期？

### 关于任务状态

2. **改任务状态**
   - 任务一开始是"还没开始"
   - 做着做着变成"进行中"
   - 做完了就"已完成"
   - 哎...但是已经完成了的任务，是不是不应该还能改回去？不然统计就乱了

### 关于提醒

3. **提醒功能**
   - 有些任务怕忘了，能设置个提醒时间
   - 到了时间就通知我一下（用控制台输出也行吧...）

### 关于统计

4. **看看学习进度**
   - 能看到我完成了多少任务
   - 还有多少没做完的
   - 完成率大概多少
   - 可以选个时间段统计，比如"这个月"或者"本周"之类的

### 关于优先级

5. **任务优先级**
   - 可以把任务设为高、中、低优先级
   - 那个...高优先级的任务是不是应该强制要求设置截止日期？不然还是不知道什么时候做完

## 一些想法（还在想的）

- 任务列表应该能全部列出来看看
- 不需要的任务应该能删掉（或者标记为删除？）
- 截止日期早于创建时间肯定不行吧...
- 已完成任务的状态不能再变回去...

## 可能会用到的数据

### 任务大概是这样的：

| 字段 | 说明 |
|------|------|
| id | 每个任务有个唯一编号 |
| title | 任务标题（必须有） |
| description | 任务描述（可以不写） |
| status | 任务状态：没开始、进行中、完成 |
| priority | 优先级：高、中、低 |
| dueDate | 截止日期（高优先级必须有） |
| userId | 我的用户ID |
| createdAt | 创建时间 |
| updatedAt | 最后修改时间 |

### 进度报告大概是这样的：

| 字段 | 说明 |
|------|------|
| reportId | 报告编号 |
| userId | 我的用户ID |
| startDate | 统计开始时间 |
| endDate | 统计结束时间 |
| totalTasks | 总共多少任务 |
| completedTasks | 完成了多少 |
| inProgressTasks | 进行中多少 |
| notStartedTasks | 还没开始多少 |
| completionRate | 完成率 |

## 一些规则

1. 截止日期不能比创建时间还早吧...这就没意义了
2. 已完成的任务不能再改回"没开始"或"进行中"，不然统计数据就乱了
3. 高优先级的任务必须有截止日期，不然怎么知道什么时候做完
