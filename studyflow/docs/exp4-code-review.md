### AI 辅助审查的价值与局限

**价值**：
- 快速扫描大量变更，发现重复逻辑（toDto 方法）
- 系统性地检查边界覆盖情况
- 辅助生成结构化的审查报告

**局限**：
- 无法理解业务特定约束（需要人工判断）
- 无法评估架构层面的影响（如 DDD 分层是否被削弱）
- 无法判断代码风格是否与团队一致
- 最终决策必须由人工做出，AI 的结论仅作参考

**人工判断的不可替代之处**：
- 评估横向影响（多个模块的一致性）
- 判断重构时机（何时该提取重复代码）
- 权衡技术债务与交付速度
- 理解业务优先级
